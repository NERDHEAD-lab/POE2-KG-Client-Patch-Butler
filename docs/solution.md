# POE2 Client Troubleshooting Guide

## 수동 진단

1. 홈페이지에서 런처를 실행합니다.
2. 작업 관리자(`ctrl` + `shift` + `ese`)를 열고 상단의 검색창에 `PathOfExile`을 검색합니다.
3. 해당 프로세스가 종료될 때 까지 대기합니다 (증상 재현)
4. 실행(`win` + `R`)을 열고 아래 명령어를 입력 후 확인합니다.
   ```
   eventvwr.msc
   ```
5. **Windows 로그** -> **응용 프로그램** 진입합니다.
6. 오른쪽 **작업** 탭에서 **현재 로그 필터링 선택**합니다.
7. 이벤트 수준에서 **경고**, **오류**를 선택하고 확인합니다.
8. 런처를 실행하고 현재 시간 사이에 오류를 확인합니다.

### CASE 1: 메모리/알 수 없는 오류
```text
모듈 이름: unknown
오류 오프셋: 0x0000000000000000
Faulting 응용 프로그램 경로: ...\Path of Exile2\PathOfExile_KG.exe
```
**원인**: 주로 메모리에 문제가 있을 경우 발생합니다. (오류 오프셋의 숫자가 0으로만 채워져있음)

**해결방법**:
초기 화면에서 **2번** 진입 후 **POE2 랜더러 캐시 초기화 선택**을 시도 했음에도 발생한다면 실제 메모리 카드의 문제를 의심 할 수 있습니다.

실행(`win` + `R`)을 열고 아래 명령어를 입력 후 확인하여 진단이 필요합니다. (많은 시간이 소요됨)
```
MdSched.exe
```

### CASE 2: 보안 프로그램 충돌
```text
Faulting 응용 프로그램 경로: ...\TENXWGuard64_127.dll
```
**원인**: 주로 금융, 관공서 보안프로그램에 의해 발생됩니다.

**예시**:
> `TENXWGuard64_127.dll`은 인터넷 뱅킹에서 사용되는 프로그램인 RaonSecure의 TouchEn key에서 사용되고 있습니다.
> **프로그램 추가/제거**에서 **TouchEn**을 검색해서 제거해주세요.
> 추후 인터넷 뱅킹과 관공서를 이용 할 때는 [**식탁보**](https://yourtablecloth.app/)를 사용하여 보안 프로그램을 격리 및 충돌을 방지 할 수 있습니다.

예시와 같이 POE2를 실행하는 시간대에 발생 하는 응용 프로그램명 혹은 모듈 이름을 검색하여 원인을 좁힐 수 있습니다.
